# Spec：使用者 Onboarding 教學

> 這份文件是 onboarding 教學的需求說明書,與 url-spec.md(URL 收藏功能)平行。
> url-spec.md 中提及的 onboarding tip 改由本 spec 規範,實作上獨立成獨立模組。
> 教學內容會隨 app 功能擴充而持續更新,本 spec 也會同步維護(見 §7 ⚠️)。

---

## 1. 目的

幫助使用者建立 app 的心智模型,降低後續踩雷的機率,特別是這幾個設計細節:

- 「資料夾 = 真相」的設計理念
- 「一次服務一個資料夾」的定位(更換資料夾 = 清快取重新開始,舊資料夾內容不動)
- 「重新整理」按鈕的雙重職責(本機檔案掃描 + 重讀 `links.md`)
- URL 收藏跟本機檔案使用兩套不同的同步邏輯
- `links.md` 和 `thumbs/` 是 app 幫使用者建在資料夾裡的,屬於使用者,可以自己搬

教學是補課性質,不是強制流程,但完全不看會踩雷(尤其外部編輯 `links.md` 的場景)。所以採取「首次出現 + 可關閉 + 可隨時重看」的折衷。

## 2. 觸發時機

### 2.1 自動跳出(首次)

- 條件:
  - 使用者第一次成功選擇資料夾
  - 首次掃描完成、首次渲染完畢
  - localStorage 中沒有 `wlib-onboarding-seen` 紀錄(或值小於當前版本,見 §3.2)
- 形式:自動跳出全螢幕遮罩 + 中央對話框

### 2.2 重新觀看(手動)

- 入口:頂欄右側的設定齒輪選單,新增一項「重看使用教學」
- 點擊後直接跳出,不需要重置 localStorage

### 2.3 不會自動跳的情境

- 後續打開 app(cache 直出,不需選資料夾)
- 更換資料夾後(同一台瀏覽器,localStorage 已標記過)
- 已標記版本 ≥ 當前版本

## 3. 形式

### 3.1 互動模式:多步驟對話框

採用 wizard 形式:

- 整個畫面被半透明遮罩蓋住,**遮罩外點擊不關閉**(避免誤觸)
- 中央一個對話框,包含:
  - 標題
  - 內容區塊(文字為主,必要時可加 icon / 小插圖)
  - 步驟指示器(例:●○○○○,顯示目前第幾步 / 共幾步)
  - 「上一步」「下一步」按鈕(第一步無上一步;最後一步的「下一步」改成「我知道了」)
  - 右上角小叉叉「跳過」(點了等同看完,localStorage 會標記為已看過)

### 3.2 版本管理

localStorage key: `wlib-onboarding-seen`,值為版本字串(例 `"v1"`,跟隨教學內容大改版升版)。

- v1 第一次推出:有值 → 不跳;無值 → 跳
- 未來內容大改版(v2):把比對邏輯擴成「localStorage 值 < 當前版本 → 跳一次升級提示」
- ⚠️ 不要每次小修都升版,避免擾民
- 升版時可考慮一個輕量的「有更新!」toast 而不是整個 wizard 重來,看內容變動程度決定

## 4. 教學內容(v1)

> 內容隨 app 功能擴充逐步補充。目前以 url-spec.md §11.1 既有內容為起點,以下是 v1 的步驟拆解。

### Step 1: 歡迎 + 設計理念

標題:「歡迎使用 Yarn Patterns Library」

內容:
```
這個 app 把你的編織資料庫變成可瀏覽的畫廊。

設計理念:
🗂 你的資料夾就是真相。
   app 只是把資料夾的內容畫給你看,
   不會搬動或修改你既有的檔案。
```

### Step 2: 兩種資料的同步方式

標題:「app 同時管兩種資料」

內容:
```
🗂 本機檔案 (PDF / 圖片)
   → 資料夾是真相
   → app 不會主動偵測檔案變動
   → 在檔案總管改了東西,要按「重新整理」同步

🔗 URL 收藏
   → URL 條目存在資料夾根目錄的 `links.md`
   → URL 縮圖存在資料夾的 `thumbs/` 子資料夾
   → 這兩個都是 app 第一次需要時自動建在你資料夾裡的,
     但它們屬於你,可以自己搬、自己看
   → 平時用 app 內按鈕新增 / 編輯 / 刪除
   → `links.md` 也是一份普通 markdown,
     可以用 VS Code 等工具直接編輯
```

### Step 3: 「重新整理」按鈕的意義

標題:「右上角的『重新整理』」

內容:
```
這個按鈕同時做兩件事:
1. 重掃資料夾的本機檔案
2. 重讀 links.md(吃進外部編輯)

✅ 何時按:
- 在檔案總管移動 / 刪除 / 改名了檔案
- 在 VS Code 改了 links.md 想立刻在 app 看到結果
- 開 app 後懷疑顯示跟資料夾不一致

🛟 不需要擔心:
外部編輯 links.md 不會造成資料遺失。
即使忘了按重新整理就在 app 內新增 URL,
你動到的條目以外,外部修改都會自動保留。
```

### Step 4: 「更換資料夾」是重新開始

標題:「換資料夾 = 重新開始」

內容:
```
這個 app 一次只服務一個資料夾。

選了新的資料夾,app 會:
✓ 清掉之前快取的封面預覽
✓ 從新資料夾重新讀取本機檔案與 links.md

🛟 舊資料夾的內容不會跟著消失:
- 你的 PDF / 圖片本來就在那裡,沒被動過
- 之前在 app 內收藏的 URL 存在
  舊資料夾根目錄的 `links.md`,
  URL 縮圖存在舊資料夾的 `thumbs/` 子資料夾
- 這兩個是 app 幫你建的,但它們屬於你
- 想把 URL 收藏帶到新資料夾?
  在檔案總管把這兩個搬過去就行
```

### Step 5: 萬一 markdown 壞了

標題:「壞檔自動還原」

內容:
```
如果 links.md 因為任何原因損毀,
app 會自動從快取救回上次成功的版本,
原本壞掉的檔案會改名成 links.md.broken-{時間},
完整保留在資料夾裡。

你不需要做任何事。
如果發現有 URL 遺失,可以到 .broken 檔裡撿回來。
```

### Step 6: 結尾

標題:「準備好了!」

內容:
```
教學結束。
如果之後想再看一次,點右上角設定齒輪 → 「重看使用教學」。

開始整理你的編織資料庫吧!
```

底部按鈕:「我知道了」(取代「下一步」)

## 5. 設計細節

### 5.1 視覺風格

- 沿用 app 既有 design system(自然冥想風,毛玻璃輕用)
- 對話框寬度:桌面 ~480px、手機左右各留邊 16px
- 對話框背景:半透明(隨深淺主題切換配色)
- ⚠️ **不要**在對話框本體用 `backdrop-filter`(按 About.md §4 hard rule,避免閃動)。遮罩用 `rgba(...)` 半透明即可

### 5.2 鍵盤支援

- Esc:跳過教學(等同點叉叉)
- ←:上一步(第一步無效)
- →:下一步 / 完成

### 5.3 無障礙

- 對話框開啟時 focus trap 在對話框內,Tab 不能跑出去
- 步驟切換時把焦點放回「下一步」按鈕
- 步驟指示器加 `aria-label`
